# GB4-MED-005..008: Talk/Broadcast, late/duplicate/early media, reconciler integration, and fake/real media-node contract suite

## Summary

Extended the GB28181 media control plane on top of GB4-MED-001..004 with sender-oriented media (two-way talk and one-way broadcast), robust handling of out-of-order / duplicate / uncertain media events, reconciler-facing behavior for restart windows, and a shared `MediaPort` contract suite that runs against both a deterministic fake media node (in CI) and, optionally, a real media node.

The signaling plane remains control-only: it negotiates SIP/SDP and drives the typed `cheetah.media.v1` / `MediaPort` boundary, and never binds RTP/RTCP ports nor parses RTP/RTCP/PS/TS/ES payloads.

## GB4-MED-005 — Talk / Broadcast and media sender capability

- `MediaPurpose::Broadcast` added alongside `Talk`; `MediaPurpose::requires_media_sender()` returns `true` for both (platform → device audio).
- `MediaRequirements::require_media_sender` propagates the sender requirement to scheduling/reservation. `build_media_requirements()` derives it from the purpose.
- `CommandPayload::StartBroadcast` and `MediaCommand::StartBroadcast` model the typed broadcast command end-to-end (domain → application → GB28181 module).
- `MediaPort::reserve_broadcast` reserves a sender-capable binding (defaults to talk-reservation semantics).
- GB28181 broadcast emits an audio `sendonly` INVITE with codec validation (PCMA/PCMU / G.711A/G.711U); unsupported codecs return a stable `MediaError::Unsupported` **before** any side effect. Talk continues to emit `sendrecv`.
- Bilateral compensation: persistence/reservation failures release the reservation; partial start failures clean up both signaling (dialog) and media (reservation/sender) sides.
- `MediaService::start_broadcast()` wires resolve → reserve → persist → dispatch, releasing the reservation on persistence failure.
- HTTP `create_session` accepts `"broadcast"` and dispatches to `start_broadcast()`.

## GB4-MED-006 — Late / duplicate / early media and terminal safety

- `MediaNodeCommandResult::UnknownOutcome { code, message }` models uncertain command results. Scheduler timeouts now map to `UnknownOutcome` rather than a terminal failure.
- Start/stop/control/reconnect handlers treat `UnknownOutcome` as non-terminal: they commit durable intent and leave the operation open for the reconciler instead of fabricating success or failure.
- Callback fencing rejects/ignores: stale signaling owner epoch, stale media-node instance epoch, wrong node identity, stale binding/session revision, and callbacks against terminal operations — none can resurrect a terminal `MediaSession`/`MediaBinding`.
- Early `Started` callbacks deterministically advance `Allocating → Inviting → Active`, persisting each revision increment in order (the in-memory repository enforces single-revision optimistic concurrency per save).
- GB28181 dialog handling covers: late 200 after CANCEL (ACK then BYE), retransmitted 200 on an active dialog (ACK without re-activation), duplicate BYE responses (harmless), device-side BYE (200 OK + terminal stop event), pending-INVITE cancellation via CANCEL (not BYE), and invalid remote SDP (ACK/BYE + failure).

## GB4-MED-007 — Reconciler integration

- Broadcast is integrated into reconnect and reconciliation/migration paths (`reserve_broadcast`, `StartBroadcast`), always creating a new generation/binding and never reviving an old terminal binding.
- `MediaPort` gained `list_nodes()` and paginated `list_sessions()` so the reconciler can detect missing media sessions and orphan media-node sessions, and validate leases / instance epochs.
- Uncertain reconnect outcomes preserve the new generation/binding for later convergence and release the new binding when convergence fails.

## GB4-MED-008 — Fake and real media-node contract suite

- New shared, transport-agnostic contract suite `cheetah_domain::media_contract::run_media_port_contract()` exercises reserve (live + broadcast), versioned reservation, `execute` start/stop, `list_nodes`, `list_sessions`, stop convergence, and idempotent release — only via the typed boundary, never touching payloads or ports.
- Deterministic fake node (`InMemoryMediaPort` + `FakeClock` + deterministic IDs + scripted results/errors) runs the suite in CI: `cheetah-domain` lib test `fake_media_node_satisfies_media_port_contract`.
- Real media-node system entry point: `crates/media/cheetah-media-scheduler/tests/media_port_system_test.rs` runs the *same* suite against a live `SchedulerMediaPort`. It is `#[ignore]`d and only executes when `CHEETAH_MEDIA_SYSTEM_ENDPOINT` is set, so default CI has no network/device dependency:

  ```text
  CHEETAH_MEDIA_SYSTEM_ENDPOINT=http://127.0.0.1:9000 \
    cargo test -p cheetah-media-scheduler --test media_port_system_test -- --ignored
  ```

## Notes / remaining work

- A durable `OperationStatus::UnknownOutcome` was intentionally **not** added: uncertain operations stay non-terminal so the reconciler owns convergence. This can be revisited if북향 queries need to distinguish "in-flight" from "uncertain".
- The generic HTTP stop route reuses the purpose-agnostic stop path; a dedicated broadcast-stop DTO can be added if product requirements diverge.
- Broadcast `MESSAGE` handshake is only emitted where the profile requires it; profile-specific handshakes can be extended as new device profiles are onboarded.

# GB4-MED-001..004: GB28181 media operation lifecycle

## Scope

GB4-MED-001..004 cover the GB28181 media control-plane state machine and its
typed command mapping:

- `GB4-MED-001` — `GbMediaMachine` in core with transaction/dialog/timer wiring
  and the live start/stop state table.
- `GB4-MED-002` — typed GB command mapper, `OperationStep`, and the MediaPort
  Open/Update/Close Saga.
- `GB4-MED-003` — UDP / TCP active-passive, SSRC / payload / setup negotiation
  and compatibility policy.
- `GB4-MED-004` — playback / download / RecordInfo / MANSRTSP / MediaStatus
  lifecycle.

Per the dependency contract in
`dev-docs/004_gb28181-improve/05_media_operations_and_reconciliation.md` §10,
until real media-node integration (MED-R / UP-MEDIA-P0) lands we deliver the
core state table, the **module command mapper**, and per-step failure tests; the
real MediaPort/driver interop (`StreamOnline`, release interop) stays `Blocked`
and must not be marked complete with fixed events.

The control plane never touches media payload: this change only produces SIP /
SDP (INVITE, ACK, BYE, CANCEL, INFO/MANSRTSP) and copies the negotiated endpoint
from the MediaPort reservation. It opens no RTP/RTCP ports and parses no
PS/TS/ES.

## State of the codebase before this task

The GB28181 media state machine (`crates/protocols/cheetah-gb28181-module/src/media/`)
already implemented:

- `Gb28181Media` sessions with `StartLive/StartPlayback/StartDownload/StartTalk`,
  `StopMediaSession` and `ControlPlayback` commands (`commands.rs`);
- INVITE/SDP offer building, ACK, BYE, CANCEL, and 200-OK / late-200 / duplicate
  handling with SDP address validation (`invite.rs`, `handlers.rs`);
- MANSRTSP INFO encoding for PLAY/PAUSE/TEARDOWN with `Scale`/`Range`
  (`control.rs`);
- UDP / TCP-passive / TCP-active transports and the `a=y:` SSRC / `a=setup`
  negotiation (`mod.rs`, `invite.rs`).

The application Saga (`crates/application/cheetah-signal-application/src/media_service*.rs`)
already implemented the Operation / MediaSession / MediaBinding lifecycle with
owner-epoch and media-node-instance-epoch fencing, MediaPort
reserve/execute/release, `StreamOnline` callbacks and reconciliation, all in one
`UnitOfWork` with the outbox.

The missing piece was the **typed mapper** that turns the already-fenced
control-plane intent (purpose, negotiated endpoint, playback control) into the
wire-facing `MediaCommand`, which is the MED-002 module mapper deliverable.

## Changes

### `crates/protocols/cheetah-gb28181-module/src/media/mapper.rs` (new)

A pure, Sans-I/O mapper from control-plane media intent to `MediaCommand`:

- `GbMediaPurpose { Live, Playback, Download, Talk }` with
  `from_domain(MediaPurpose)`. The control-plane `MediaPurpose` collapses
  playback and download into `Playback`; the GB module keeps `Download` explicit
  because the wire behaviour differs (`s=Download` + `a=downloadspeed`).
- `GbSipRouting` (target, device id, Call-ID, local tag, CSeq, branch, subject)
  and `GbMediaEndpoint` (media address/port, `a=y:` SSRC, transport) — every
  physical value is supplied by the caller from the MediaPort reservation; the
  mapper invents none.
- `GbStartRequest` + `map_start` — validates the structural invariants of each
  purpose and returns:
  - `MediaError::InvalidState` for internally inconsistent requests (missing
    recording window for playback/download, missing video SSRC, download speed on
    a non-download purpose, zero download speed, audio codec on a non-talk
    purpose, video SSRC on talk);
  - `MediaError::Unsupported` for capabilities outside the v1 contract
    (unknown talk codec, `MediaPurpose::Unknown`) — no transcoding, no fallback
    to an undeclared codec.
- `map_control(MediaSessionId, &MediaControl)` — maps the domain
  `MediaControl` to `ControlPlayback`:
  - `Play`→PLAY, `Pause`→PAUSE, `Stop`→TEARDOWN;
  - `Seek { offset_ms }`→PLAY with `Range: npt=<seconds>-` (negative offset
    rejected);
  - `Scale { value }`→PLAY with `Scale: <value>` (non-finite rejected).

The new types are re-exported from `media/mod.rs` and the crate root.

### Tests and golden fixtures

`crates/protocols/cheetah-gb28181-module/src/media/tests/mapper_tests.rs` (new):

- End-to-end golden fixtures: `map_start` → `Gb28181Media::process` →
  `encode_message`, compared against committed wire captures under
  `src/media/tests/golden/`:
  - `live_invite.sip`, `playback_invite.sip`, `download_invite.sip`,
    `talk_invite.sip`;
  - `control_play.sip`, `control_pause.sip`, `control_seek.sip`,
    `control_scale.sip` (in-dialog MANSRTSP INFO after the session is driven to
    Active). Regenerate with `UPDATE_GOLDEN=1`.
- Per-step validation / unsupported-capability tests for every rejected mapping
  branch and the domain `MediaPurpose`/`MediaControl` mappings.

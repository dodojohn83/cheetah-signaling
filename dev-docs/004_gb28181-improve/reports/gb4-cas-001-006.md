# GB4-CAS-001..006：GB28181 级联平台模型（注册/退避、多上级、目录/订阅/桥接、loop/ACL）

## 任务目标

实现 GB28181 级联平台的控制面模型与信令编排：

- `GB4-CAS-001`：`GbPlatformLink` 聚合与仓储（tenant、平台身份、方向、realm/endpoint、凭据引用、ACL、订阅状态、owner/generation/revision），SQLite/PostgreSQL 共用 contract。
- `GB4-CAS-002`：上级 REGISTER/keepalive/deregister，指数退避 + jitter，凭据按引用解析。
- `GB4-CAS-003`：目录查询/通知与分页（复用单链路状态机）。
- `GB4-CAS-004`：SUBSCRIBE/NOTIFY 容量与刷新（复用单链路状态机）。
- `GB4-CAS-005`：live/playback 桥接与 `MediaPort` 边界（复用单链路状态机）。
- `GB4-CAS-006`：多上级隔离、下级接入、平台身份校验、ACL、loop/hop、唯一 control owner。

信令层不处理媒体负载；桥接协商经 `MediaPort`。

## 实现位置

### 领域聚合（GB4-CAS-001）

- `crates/foundation/cheetah-signal-types/src/id.rs`：新增 `PlatformLinkId`（UUIDv7），并在 `ports.rs`/`test_support.rs`/各 `IdGenerator` 实现中增加 `generate_platform_link_id`。
- `crates/domain/cheetah-domain/src/platform_link.rs`：新增 `GbPlatformLink` 聚合，字段全部私有，经方法维护不变量并递增 `Revision`。
  - 值对象：`PlatformDirection`、`DesiredRegistrationState`、`ActualRegistrationState`、`PlatformIdentityPair`、`PlatformEndpoint`、`PlatformCredential`（仅引用，无明文）、`PlatformAcl`、`BackoffPolicy`、`SubscriptionLimits`、`RegistrationRuntime`。
  - 行为：`assign_owner`（owner epoch fencing + generation bump）、`is_current_owner`、注册/keepalive/退避状态迁移、`reserve/release_subscription`（有界容量）、`is_expired`、`would_loop`。
  - 顶层 `detect_loop` + `MAX_CASCADE_HOPS` 供模块层桥接路由复用。
- `crates/domain/cheetah-domain/src/ports.rs`：新增 `PlatformLinkRepository`（`get`、`get_by_remote_identity`、`save`、`delete`、`list`，全部显式 `TenantId`）。
- `crates/domain/cheetah-domain/src/in_memory.rs`：`InMemoryPlatformLinkRepository`。

### 存储（GB4-CAS-001）

- `migrations/sqlite/0010__platform_links.sql`、`migrations/postgres/0010__platform_links.sql`：追加式迁移，聚合存 `data` JSON，索引字段单列，`revision` 乐观并发。
- `crates/storage/cheetah-storage-sqlite/src/platform_link.rs`、`crates/storage/cheetah-storage-postgres/src/platform_link.rs`：读写池分离、tenant 谓词、`WHERE revision = EXCLUDED.revision - 1` 乐观并发、稳定游标分页。
- `cheetah-storage-api` 的 `Storage` trait 增加 `platform_link_repository()`，SQLite/PostgreSQL 均实现。

### 多链路管理器（GB4-CAS-006 + 002/003/004/005 接线）

- `crates/protocols/cheetah-gb28181-module/src/cascade/manager.rs`：`CascadeManager`。
  - `map_config`：将 `GbPlatformLink` 映射为 `CascadeConfig`（退避、注册间隔、订阅上限来自聚合，是持久化控制面的唯一真值来源）。
  - 每个上级持有独立 `Gb28181Cascade`（独立 Call-ID/CSeq/auth/state）；`process`/`tick_all` 按链路隔离，单链路错误不影响其他链路。
  - 下级接入 `enroll_downstream` 与平台身份校验 `validate_platform_identity`（平台身份独立于设备/通道身份）。
  - ACL 门禁：`may_share_catalog`/`may_control`/`may_bridge`。
  - 桥接路由 `authorize_bridge`：loop 检测（复用 `detect_loop`）、hop 上限、denied 平台、media ACL。
  - 唯一 control owner：`acquire_control`/`release_control`（对同一 owner 幂等，对不同 link 冲突）。
  - 新增 `CascadeRoutingError`。

单链路的 REGISTER/keepalive/deregister（CAS-002）、Catalog query/notify + 分页（CAS-003）、SUBSCRIBE/NOTIFY 容量/刷新（CAS-004）、桥接 saga + MediaPort（CAS-005）沿用既有 `Gb28181Cascade` 状态机，管理器在其之上叠加多链路与跨链路策略。

## 测试

- 领域：`platform_link.rs` 单测覆盖身份校验、owner epoch fencing、退避超限断连、订阅容量、ACL、loop 检测、确定性退避。
- 存储 contract：`crates/testing/cheetah-storage-tests/src/contract/platform_link.rs` + `fixtures.rs::platform_link`，覆盖 CRUD、`get_by_remote_identity` 方向作用域、tenant 隔离、乐观并发、delete 语义、方向过滤 + 游标分页；SQLite 与 PostgreSQL 同一 suite。
- 管理器：`cascade/manager/tests.rs` 覆盖多上级独立注册、重复 remote/自环拒绝、未知链路隔离、下级接入 + 身份校验、ACL 门禁、loop/hop、media ACL、唯一 control owner 幂等与冲突。

## 边界与后续

- 未在信令层处理任何 RTP/RTCP/PS/TS/ES；桥接协商仍走 `MediaPort`。
- core/driver/module 分层未合并；`domain` 未引入 Tokio/SQLx/协议类型。
- 后续可将持久化 `GbPlatformLink` 与应用层 reconciler/Operation 打通，由 owner 节点驱动 `CascadeManager` 的注册/桥接生命周期，并补充跨节点 owner takeover 的集成测试。

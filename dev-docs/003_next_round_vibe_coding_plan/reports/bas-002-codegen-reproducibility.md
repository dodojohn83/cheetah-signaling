# BAS-002: Proto Codegen Reproducibility

Date: 2026-07-22

## Summary

The protobuf Rust codegen for `cheetah-signal-contracts` is reproducible from a clean target. Two separate builds from empty `CARGO_TARGET_DIR` directories produced byte-identical generated `.rs` files.

## Script

`scripts/verify_proto_codegen_reproducible.py` implements the check:

1. Create two temporary `CARGO_TARGET_DIR` directories.
2. Run `cargo check -p cheetah-signal-contracts` in each.
3. Locate the `OUT_DIR` of each build (`target/debug/build/cheetah-signal-contracts-*/out`).
4. Compare the generated files by path and content.
5. Exit `0` only if all files are byte-identical.

## Result

```text
OK: 7 generated file(s) are byte-identical across two clean builds.
OUT_DIR run-1: .../a/debug/build/cheetah-signal-contracts-901964d56242caf9/out
OUT_DIR run-2: .../b/debug/build/cheetah-signal-contracts-901964d56242caf9/out
```

The build directories use the same stable hash, and the seven `.rs` files generated by `tonic-build`/`prost-build` match exactly. This satisfies the BAS-002 requirement that codegen from an empty target is executable and produces no diff between runs.

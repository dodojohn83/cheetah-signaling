# Cheetah Signaling Protobuf Contracts

Versioned cross-process and persistence wire schema.

- `cheetah/common/v1/` — shared types, IDs, envelope, errors and pagination
- `cheetah/device/v1/` — device domain snapshot and device events
- `cheetah/control/v1/` — control commands, results and `NodeCommand` service
- `cheetah/plugin/v1/` — plugin SDK runtime frame and `PluginRuntime` service
- `cheetah/media/v1/` — media DTO, media commands/events and media services
- `cheetah/cluster/v1/` — node and media-node registry, cluster events

Managed by `buf` v2. Use `buf lint` and `buf breaking --against '.git#branch=main'` before merging.

## Toolchain versions

The following versions are canonical for this workspace and must be used in CI, development containers and local installs:

- `protoc` **25.3** (download: https://github.com/protocolbuffers/protobuf/releases/tag/v25.3)
- `buf` **1.50.0** (download: https://github.com/bufbuild/buf/releases/tag/v1.50.0 or `go install github.com/bufbuild/buf/cmd/buf@v1.50.0`)

## Reproducible codegen

The Rust bindings are generated by `crates/foundation/cheetah-signal-contracts/build.rs` into `OUT_DIR`. To verify that codegen is deterministic from a clean target:

```bash
# From the workspace root
rm -rf target
PROTOC=/path/to/protoc cargo check -p cheetah-signal-contracts
# Run a second time; OUT_DIR contents should be byte-identical
```

CI pins these exact versions in `.github/workflows/ci.yml`.
